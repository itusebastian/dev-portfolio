Migración a GitHub Pages

De WordPress a GitHub Pages

Siempre he sido un apasionado de la escritura, y mi blog de ficción ha sido una parte importante de mi vida durante más de diez años. RelatoCorto.com ha sido el espacio donde compartía historias, donde mi creatividad tenía un hogar. Sin embargo, recientemente tomé una decisión importante: embarcarme en una nueva aventura en el mundo del desarrollo de software.

Este cambio no ha sido fácil. Para poder enfocarme en mis estudios de programación, decidí recortar gastos y, con pesar, tuve que cancelar mi servicio de hospedaje en WordPress. Fue una decisión difícil, ya que mi blog había estado ahí durante más de una década, pero sabía que necesitaba priorizar. En este proceso, descubrí algo maravilloso: GitHub Pages.

GitHub Pages es un servicio gratuito que te permite publicar contenido de un repositorio en GitHub y mostrarlo en un dominio propio. En ese momento, supe que debía intentarlo. Me dediqué durante días a construir el nuevo sitio web desde cero, utilizando HTML, CSS y JavaScript. Quería que mi página funcionara como un CMS (Sistema de Gestión de Contenidos), pero creado por mí, sin interfaces complicadas o limitaciones. Mis conocimientos actuales de JavaScript me ayudaron a lograrlo: ahora tengo un CMS personalizado para desarrolladores, sin una interfaz visual, pero perfectamente funcional.

Lo mejor de todo es que RelatoCorto.com ya tiene un nuevo hogar. Aún no sé qué pasará mañana, pero por ahora, el sitio está funcionando como lo imaginé, y esa sensación es increíble.

Mientras espero el inicio de las clases en la universidad, he decidido aprovechar este tiempo ayudando a algunos clientes a respaldar sus sitios web estáticos. Incluso he convertido varios de sus sitios de WordPress a HTML y los he subido a GitHub Pages, lo que les permite alojar sus páginas de forma gratuita. Me siento motivado al ver cómo todo empieza a dar frutos.

Solo han pasado unos meses desde que empecé a estudiar programación, y ya entiendo por qué muchos desarrolladores prefieren evitar herramientas como WordPress. Tener control total sobre el código es un cambio radical. La libertad de poder editar directamente el HTML, CSS o JavaScript sin depender de plantillas o plugins es algo que ahora aprecio mucho más.

Este mes ha sido uno de los más importantes en mi viaje de transición de autor a desarrollador. He aprendido mucho, no solo sobre desarrollo web, sino sobre la importancia de adaptarse y crecer. RelatoCorto.com ha sobrevivido a este cambio y está más vivo que nunca. Estoy emocionado por lo que vendrá, tanto en el mundo de la escritura como en el desarrollo de software.

¿Quién diría que la creatividad y la tecnología se pueden fusionar tan bien?
